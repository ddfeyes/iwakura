# Spec: Fix /api/status hanging due to N sequential tmux capture-pane calls
**Issue:** #61
**Date:** 2026-03-16
**Priority:** P0 (platform blocker)

## Problem
`get_ao_sessions()` in `backend/status.py` calls `tmux capture-pane -t <session> -p`
for every iw- tmux session created in the last 24h. With 27+ sessions, this is 27+
sequential async subprocess calls (5s timeout each), making /api/status hang.

## Fix Spec

### Change 1: Reduce window from 24h → 2h
Active work only happens in sessions created within the last ~2 hours.
Sessions older than 2h are done; count them as `old_count` instead.

```python
THRESHOLD_RECENT = 7200  # 2 hours instead of 86400
```

### Change 2: Concurrent capture-pane via asyncio.gather
Instead of sequential `await _run(capture-pane)` per session, gather all at once:

```python
# Collect all session names first, then gather
names = [s["name"] for s in candidate_sessions]
captures = await asyncio.gather(*[
    _run(["tmux", "capture-pane", "-t", name, "-p"])
    for name in names
])
```

### Change 3: Cap at 5 most recent sessions
Even with concurrent captures, limit to 5 newest sessions to avoid overhead.
Sort by `created_ts` descending, take first 5.

## Files to Change
- `backend/status.py` — `get_ao_sessions()` function only
