# Spec: Glitch Effects — CRT Scanlines, Screen Flicker, Data Corruption Aesthetic
**Issue:** #26  
**Date:** 2026-03-16

## Goal
Enhance `frontend/js/effects.js` with a comprehensive glitch/CRT visual effects system matching the PSX Lain aesthetic. All effects must be toggleable and 60fps-target on MacBook.

---

## Effects Inventory

### 1. CRT Scanlines (configurable)
- CSS-only overlay via `::after` pseudo-element or a dedicated `<div class="crt-overlay">`
- Properties: `opacity`, `lineHeight` (gap between lines), `animated` (subtle scroll)
- Respect `prefers-reduced-motion`: disable scroll animation, reduce opacity
- Default: `opacity: 0.08`, `lineHeight: 2px`, slow 8s scroll

### 2. Screen Flicker
- Random interval 3–8s, very brief (80–120ms) brightness pulse via CSS filter
- Implementation: `GlitchEffects.flicker()` — sets a CSS class `.flicker` on `<body>`, removes after timeout
- CSS: `.flicker { animation: flicker-pulse 100ms forwards }`
- `@keyframes flicker-pulse`: brightness 1 → 1.4 → 0.7 → 1
- Reduced motion: disable entirely

### 3. Text Glitch (character corruption)
- Target: `[data-glitch]` elements (headers, codes, terminal lines)
- Randomly replace 1–3 chars with katakana/hex chars for 80–200ms, then restore
- Trigger: random interval 4–12s per element, OR on class `.glitch-trigger` add
- CSS `data-glitch` attribute approach: `element.dataset.originalText` stored on init
- Reduced motion: skip

### 4. Color Aberration (RGB channel split)
- On hover/transition: brief 1–2px offset of red+blue channels
- CSS: `.aberration { text-shadow: -1px 0 rgba(255,0,0,0.4), 1px 0 rgba(0,0,255,0.4) }`
- Apply on `mouseenter`, remove after 300ms
- Also auto-trigger on screen transitions (add class during navigation)
- Reduced motion: no auto-trigger, hover OK

### 5. Boot Sequence Enhancement
- Matrix-style reveal: characters cycle randomly 4–6 times before snapping to final char
- Per-character timing: `charDelay * index` (staggered, 20ms per char)
- After all chars resolved: 150ms pause, then remove cycling class
- Replace existing typewriter in `effects.js` or add `matrixReveal(el, text, opts)` function
- Reduced motion: instant reveal

### 6. Psyche Screen Glitch Flash
- On data refresh in psyche screen: add `.glitch-flash` class to psyche container
- `@keyframes glitch-flash`: brief horizontal translate (±3px) + hue-rotate (±20deg) × 3 cycles, 200ms total
- Trigger: hook into psyche data refresh call (or expose `GlitchEffects.psycheFlash(el)`)

### 7. Data Rain Optimization (offscreen canvas)
- Pre-render all katakana characters onto an offscreen `OffscreenCanvas` (or regular off-DOM canvas)
- Cache character images as `ImageBitmap` or pre-drawn tiles at each font size
- In `_frame()`, use `ctx.drawImage()` for pre-rendered char tiles instead of `fillText()`
- Target: reduce per-frame CPU by ~40% on dense matrices

## API Design

```js
// Global singleton
const glitchFX = new GlitchEffects({ reducedMotion: prefersReducedMotion() });

glitchFX.enableScanlines(opacity = 0.08);
glitchFX.disableScanlines();
glitchFX.startFlicker();
glitchFX.stopFlicker();
glitchFX.enableTextGlitch(selector = '[data-glitch]');
glitchFX.disableTextGlitch();
glitchFX.psycheFlash(containerEl);
glitchFX.bootReveal(containerEl, callback);
```
