# Spec: Keyboard Shortcuts / Hotkeys (Issue #34)
Date: 2026-03-16

## Problem
No keyboard navigation. Platform feels consumer-app, not terminal/hacker.

## Solution
Add `hotkeys.js` module with a single `keydown` listener, PSX help overlay.

## Implementation

### hotkeys.js (new file)
```js
class IwakuraHotkeys {
  constructor(appShowScreen, audioToggleMute) { ... }
  init() { document.addEventListener('keydown', this._onKey.bind(this)) }
  _isTyping() { return ['INPUT','TEXTAREA'].includes(document.activeElement?.tagName) }
  _onKey(e) {
    if (this._isTyping()) return;
    // screen shortcuts
    const map = { '1': 'diary', '2': 'status', '3': 'memory', '4': 'psyche' };
    if (map[e.key]) { appShowScreen(map[e.key]); return; }
    if (e.key === 'Escape') { appShowScreen('hub'); return; }
    if (e.key === 'm') { audioToggleMute(); return; }
    // diary-only
    if (currentScreen === 'diary') {
      if (e.key === '/') { e.preventDefault(); focusDiaryInput(); return; }
    }
    // help
    if (e.key === '?') { toggleHelpOverlay(); return; }
  }
}
```

### Help overlay
- `<div id="hotkeys-help" class="hotkeys-overlay hidden">` in index.html
- Table of shortcuts, PSX-styled
- `<button id="hotkeys-btn" class="hotkeys-btn">[ ? ]</button>` fixed bottom-left

### psx.css additions
- `.hotkeys-overlay` — dark translucent panel, centered, monospace, orange headers
- `.hotkeys-btn` — fixed bottom-left, small, orange text button, z-index high
- `.hotkeys-table` — two-column, green text for keys, white for actions

### app.js integration
- After boot completes: `window.hotkeys = new IwakuraHotkeys(showScreen, toggleMute); hotkeys.init();`
- Export `currentScreen` variable hotkeys.js can read
