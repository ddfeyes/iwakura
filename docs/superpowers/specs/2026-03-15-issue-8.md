# Spec: PSX CSS System — Issue #8

**Date:** 2026-03-15  
**Issue:** #8 — feat: PSX CSS system — scanlines, CRT, data rain, palette  
**Status:** APPROVED (autonomous)

## Goal

Create a reusable, global CSS system that gives every Iwakura screen the PSX/Serial Experiments Lain aesthetic. This is the visual foundation all other screens depend on.

## Deliverables

### 1. `frontend/static/css/psx.css` — Core system file

**Color palette (CSS custom properties):**
```css
--color-bg: #0a0a1a
--color-bg2: #0d0d2b
--color-accent-orange: #ff8c00
--color-accent-cyan: #00d4aa
--color-accent-purple: #8b7cc8
--color-text: #e0e0e0
--color-text-green: #00ff88
--color-blue-dot: #3a7bd5
```

**Typography:**
- Import VT323 + Share Tech Mono from Google Fonts
- `.font-psx` → VT323, monospace
- `.font-mono` → Share Tech Mono, monospace
- Default body font: Share Tech Mono

**Scanlines overlay:**
- `.scanlines` class — pseudo-element with repeating linear gradient, 2px lines, opacity ~0.08
- Applied to `body` or wrapping container

**Data rain background:**
- `.data-rain` class — canvas-based OR CSS animation with katakana chars
- Very subtle, opacity ~0.04, positioned fixed behind content

**Progress bars (blue dots):**
- `.progress-dots` — a row of dot indicators, filled blue vs empty
- Example: `● ● ● ○ ○` (3/5)

**Dividers:**
- `.hr-orange` — 1px solid #ff8c00, with slight glow

**Notification style (gate pass):**
- `.gate-pass` — small monospace box, orange border, dark bg, status indicator

**Utility classes:**
- `.text-orange`, `.text-cyan`, `.text-green`, `.text-purple`
- `.bg-psx` — dark navy background
- `.border-orange`, `.border-cyan`
- `.glow-orange`, `.glow-cyan` — text-shadow glow effects

### 2. `frontend/static/js/data-rain.js` — Canvas data rain

- Creates a `<canvas>` element fixed to background
- Animates falling katakana/ASCII chars in green (#00ff88) at very low opacity
- Respects `prefers-reduced-motion`

### 3. Update `frontend/templates/base.html` (if exists) or `index.html`

- Link `psx.css`
- Include `data-rain.js`
- Add `.scanlines` to body

## Acceptance Criteria

- [ ] All screens automatically get PSX aesthetic when `psx.css` is loaded
- [ ] Scanlines visible but not distracting (opacity ≤ 0.1)
- [ ] Data rain visible but subtle (opacity ≤ 0.05)
- [ ] Color variables usable via `var(--color-*)` in other CSS files
- [ ] No JS framework dependencies
- [ ] Works on localhost:8790

## Notes

- This CSS should be imported FIRST before any screen-specific CSS
- Data rain canvas should be z-index: -1
- All fonts loaded from Google Fonts CDN
