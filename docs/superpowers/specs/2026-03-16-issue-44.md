# Spec: STATUS — Fix OpenClaw Crons + Compact Idle AO Sessions (Issue #44)

## Problem
1. GET /api/crons on OpenClaw gateway returns 404 → openclaw_crons always empty
2. STATUS screen shows 18 idle AO sessions cluttering the view

## Solution

### Part 1: Fix OpenClaw Crons (backend/status.py)

Try these endpoints in order (the OpenClaw gateway doesn't expose a public REST API for crons — use CLI instead):

```python
async def get_openclaw_crons() -> list[dict]:
    """Fetch cron jobs via openclaw CLI."""
    try:
        import json
        proc = await asyncio.create_subprocess_exec(
            "openclaw", "cron", "list", "--json",
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        stdout, _ = await asyncio.wait_for(proc.communicate(), timeout=5.0)
        data = json.loads(stdout.decode())
        jobs = data if isinstance(data, list) else data.get("jobs", [])
        result = []
        for item in jobs:
            result.append({
                "name": item.get("name", ""),
                "schedule": str(item.get("schedule", "")),
                "enabled": item.get("enabled", True),
                "last_run": item.get("lastRun", item.get("last_run", "")),
            })
        return result
    except Exception:
        return []
```

If `openclaw cron list --json` doesn't work, try:
- `openclaw cron list` and parse text output
- Read ~/.openclaw/openclaw.json for cron config

### Part 2: Compact Idle AO Sessions (frontend/js/status.js)

In the AO sessions rendering section:
- Classify session: active if `last_line` does NOT contain 'bypass permissions' AND age_seconds < 1800
- Idle otherwise
- Render active sessions fully (existing card style)
- Collapse idle into a single row: `🟡 N IDLE SESSIONS (click to expand)`
- On click toggle, show idle sessions in dim style
