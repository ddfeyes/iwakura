# Spec: Three.js Orbital Hub Navigation (Issue #24)
Date: 2026-03-16
Issue: #24

## Goal
Replace the current 2D hub canvas with a Three.js orbital navigation system that matches the PSX game aesthetic.

## Visual Design
- 2-3 elliptical rings, each rotated at different axes (e.g. rotateX 25°, rotateY 45°, rotateZ 15°)
- Rings rotate slowly in opposite directions (0.001-0.003 rad/frame)
- 4 bubble nodes placed on rings: DIARY (Lda020), STATUS (Tda040), MEMORY (Wld033), PSYCHE (Ira007)
- Each node: translucent blue-gray sphere with orange label beneath
- Hover: node scale 1.3x + cyan glow halo
- Lain center: glowing sphere, color #8b7cc8, pulse animation (scale 0.95-1.05, 2s sine)
- Background: existing data rain canvas stays (behind Three.js canvas)

## Colors
- Ring: #1a1a3e, opacity 0.6
- Node: #2a2a5e with orange (#ff8c00) border glow
- Lain node: #8b7cc8 
- Selected/hover: cyan (#00d4aa) glow

## Interaction
- Click bubble → call app.navigateTo(target) where target = 'diary'|'status'|'memory'|'psyche'
- Hover → cursor: pointer + node scales up
- ESC → go back to hub (already handled by app.js)

## Implementation
- Three.js from CDN: https://cdn.jsdelivr.net/npm/three@0.160.0/build/three.module.js
- Mount into existing #hub-canvas element (or overlay canvas if needed)
- Must clean up on screen leave (dispose renderer, cancel animation frame)
- Keep all existing HTML nav-label-item elements but hide them visually (Three.js replaces them)
- File: frontend/js/nav.js — replace or extend

## Tests
- Manual: hub screen shows 3D rings, nodes clickable, transitions work
- CI: backend-check and frontend-check pass (no Python/lint issues)

## Acceptance
- Rings visible and rotating
- All 4 nodes clickable and navigate to correct screen
- Lain center node pulses
- No console errors
- Performance: smooth 60fps on MacBook

## Constraints
- No npm, no build step
- Must work with existing app.js screen management
- Vanilla JS module pattern
