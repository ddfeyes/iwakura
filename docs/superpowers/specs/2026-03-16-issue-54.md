# Spec: STATUS — Kill Idle AO Sessions + Auto-hide Old Sessions

**Issue:** #54
**Date:** 2026-03-16
**Status:** approved (autonomous)

## Problem
STATUS screen shows 23 idle AO sessions (iw-1 through iw-23). All show "bypass permissions" which means idle Claude Code prompt. No value in displaying them.

## Implementation Plan

### backend/status.py
1. `get_ao_sessions()`:
   - Read session creation time from tmux `#{session_created}` (already in format)
   - Add `age_hours: float` field
   - Return ALL sessions but flag old ones
   - Filter: sessions where `age_seconds > 86400` (>24h) → return summary entry only

2. New FastAPI route `DELETE /api/ao-sessions/idle`:
   ```python
   @app.delete("/api/ao-sessions/idle")
   async def kill_idle_sessions():
       sessions = await get_ao_sessions()
       killed = []
       for s in sessions:
           is_idle = 'bypass permissions' in (s.get('last_line') or '').lower()
           old_enough = (s.get('age_seconds') or 0) > 3600
           if is_idle and old_enough:
               await _run(['tmux', 'kill-session', '-t', s['name']])
               killed.append(s['name'])
       return {"killed": killed, "count": len(killed)}
   ```

### frontend/js/status.js
1. AO SESSIONS card header: add `[KILL IDLE ✕]` button (orange, monospace)
2. On click: `fetch('/api/ao-sessions/idle', {method:'DELETE'})` → then `this.refresh()`
3. Show toast: `✓ N sessions terminated`
4. In session list: sessions with `age_seconds > 86400` → show as `14 sessions >24h (hidden)`
5. Idle sessions collapsed by default (already done), but add count badge

### CSS (frontend/css/style.css)
- `.kill-idle-btn` — orange border, transparent bg, hover: red border + text
- `.sessions-hidden-badge` — dim gray, small monospace text
