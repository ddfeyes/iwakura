# Spec: Streaming Chat Responses (Issue #18)

## Problem
Chat currently shows full Lain response only after 10-30s wait. Feels dead.

## Solution
Stream tokens from OpenClaw CLI → WebSocket → frontend in real-time.

## Backend (gateway.py)
```python
async def stream_message(text: str, session_id: str | None) -> AsyncIterator[str]:
    """Stream response tokens from openclaw agent CLI."""
    cmd = ["openclaw", "agent", "--agent", "lain", "--message", text]
    if session_id:
        cmd.extend(["--session-id", session_id])
    # Run without --json, capture stdout line by line
    proc = await asyncio.create_subprocess_exec(*cmd,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE)
    async for line in proc.stdout:
        text = line.decode().rstrip('\n')
        if text:
            yield text
    await proc.wait()
```

## WebSocket Protocol (main.py)
- On recv `{type: "message"}` → start streaming
- Send `{type: "token", text: chunk}` for each token
- Send `{type: "done", sessionId: "..."}` when complete
- Keep `{type: "thinking"}` at start

## Frontend (chat.js)
- On `type: "token"`: if no current Lain bubble → create one; append text to it
- On `type: "done"`: finalize bubble, update session indicator
- Keep typewriter cursor while streaming

## Test
- Send a message, first characters should appear within 2s
- Long response streams token-by-token smoothly
- UI shows ▋ cursor while streaming

## Notes
- If openclaw agent only supports --json (no streaming), fallback: run with --json,
  buffer full response, then send as one token (no streaming but no regression)
- Preferred: check if openclaw supports --stream flag first
