# Spec: STATUS — AO tmux sessions + OpenClaw cron jobs (Issue #42)

## Context
The STATUS screen in the Iwakura PSX platform currently reads cron jobs from `crontab -l`
which returns near-empty results on this machine. AO session states are entirely missing.
The SPEC explicitly requires both. This issue adds them.

### frontend/js/status.js changes

#### AO Sessions section (new)
- Heading: "AO SESSIONS" in orange
- For each session: `● <name>  <age>  <last_line_truncated>`
- Dot color: green if active, gray if idle
- If no sessions: `● NO ACTIVE SESSIONS`

#### Cron section replacement
- Replace the `crons` section renderer
- Use `openclaw_crons` array instead of `crons`
- Heading: "CRON JOBS" in orange  
- For each job: `◉ <name>  <schedule>  [ENABLED|DISABLED]  <last_run>`
- Blue dot (◉) for enabled, gray for disabled
- If no crons: show `◉ NO CRON DATA`
- Keep fallback to old `crons` array if `openclaw_crons` missing

## Acceptance Criteria
1. `GET /api/status` response contains `ao_sessions: [...]` and `openclaw_crons: [...]`
2. STATUS screen shows "AO SESSIONS" section with live tmux data
3. STATUS screen shows "CRON JOBS" section using OpenClaw gateway data
4. Colored dots match PSX aesthetic (green/gray for AO, blue for crons)
5. No JS errors if either data source is missing/empty
6. Backend handles tmux/gateway unavailability without crashing

## Notes
- OpenClaw gateway runs at 127.0.0.1:18789
- AO sessions follow pattern `*-iw-N` (e.g., `b7e853f412b9-iw-17`)
- Use existing PSX CSS classes: `.status-dot`, `.green`, `.cyan`, `.orange`
- Age display: "2h 3m", "45m", "< 1m"
