# Spec: Memory Browser — Issue #20

Date: 2026-03-16
Issue: https://github.com/ddfeyes/iwakura/issues/20

## Problem
Memory screen exists in nav but is not implemented. `/api/memory` endpoints exist but frontend ignores them.

## Goal
A working memory browser that displays Lain's actual memory files with PSX aesthetic — feels like browsing private files in a retro OS.

## Implementation

### File: `frontend/js/memory.js`
- `initMemory()` — called when memory screen enters view
- Fetch `GET /api/memory` → returns list of `{ id, name, size, modified }` objects
- Render as scrollable list with PSX style:
  - File code (e.g. `Mda001`), name, size in KB, date
  - Selected item highlighted (green-on-black or amber)
- On select (Enter or click): fetch `GET /api/memory/{id}` → render content
- Content view: plain text, monospace, scrollable, markdown stripped or displayed as-is
- Navigation: arrow keys up/down to move selection, Enter to open, Esc to return to list
- Back button returns to hub screen

### Integration (`frontend/js/app.js`)
- Import and call `initMemory()` when memory screen becomes active
- Handle Esc key in memory screen context

### Styling (`frontend/css/style.css`)
- Memory list: consistent with existing PSX panel style
- Selected row: bright highlight (e.g. `#00ff00` text on dark bg)
- Content panel: separate scrollable area, monospace font
- Cursor blink on selected item

## Acceptance Criteria
- Memory screen shows real file list from `/api/memory`
- Each file can be read by selecting it
- Keyboard navigation (up/down/enter/esc) works
- Back/Esc returns to hub
- Aesthetic matches PSX game theme
