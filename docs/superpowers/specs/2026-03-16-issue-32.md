# Spec: Diary Message Persistence (Issue #32)
Date: 2026-03-16

## Problem
DIARY screen loses all chat history on page refresh. Every session starts blank.

## Solution
Persist last 100 messages to `localStorage` and restore on init.

## Implementation

### Storage schema
```js
// Key: iwakura-diary-history
// Value: JSON array (max 100, FIFO)
[
  { role: 'user'|'lain', text: string, timestamp: string, fileCode: string },
  ...
]
```

### chat.js changes
1. `init()` — after container set, before WebSocket connect:
   - Read + parse `iwakura-diary-history`
   - If valid array + non-empty → render each message using existing render methods
   - Insert separator: `── PREVIOUS SESSION ──` (div.session-sep)
   - On WebSocket connect: insert `── CURRENT SESSION ──` separator
2. After user message renders → `_saveMessage({ role:'user', text, timestamp, fileCode })`
3. After Lain `done` event → `_saveMessage({ role:'lain', text: this._streamBuf, timestamp, fileCode })`
4. `_saveMessage(msg)`:
   - Read array, push msg, trim to last 100, write back
   - Wrap in try/catch — silently skip on quota error or bad JSON

### index.html changes
- Add `<button id="clear-diary-btn" class="diary-clear-btn">[ CLEAR DIARY ]</button>` in diary screen header area
- Wire up in app.js or chat.js

### Clear diary flow
- On button click: inject into chat container:
  ```
  div.purge-prompt: "> CONFIRM PURGE? TYPE Y + ENTER OR ESC TO CANCEL"
  + input.purge-input (autofocused)
  ```
- If user types `y` or `Y` + Enter: clear localStorage key + clear container + show `> DIARY CLEARED`
- If Escape: remove prompt, do nothing
- Not a browser alert — inline PSX terminal style

### psx.css additions
- `.session-sep` — faint horizontal rule, green text `──` centered, 0.6 opacity
- `.diary-clear-btn` — small orange text button, top-right of diary header, monospace
- `.purge-prompt` — orange text prompt line in chat area
- `.purge-input` — minimal orange underline input
