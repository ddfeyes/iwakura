# WIRED screen — live activity stream + system telemetry

**Issue:** #58 (to be created)
**Type:** enhancement, frontend, backend
**Priority:** P1

## Problem
Iwakura has 6 screens (DIARY, STATUS, MEMORY, PSYCHE, TASKS, SEARCH) but no way
to see Lain's "pulse" — what's happening right now across the system. The hub feels
static between interactions.

## Solution
Add a 7th orbital node: **WIRED** — a live activity stream in PSX terminal aesthetic.
It streams recent events from multiple sources into a scrolling feed:
- Diary messages (sent/received, last 10)
- AO session events (spawned, completed, idle)
- Memory writes (file modified, size delta)
- Cron job runs (triggered, result)
- GitHub activity (recent commits from /api/github if available, else skip)
- System alerts (memory pressure, docker status changes)

## Backend

### New endpoint: GET /api/wired
Returns last 50 events sorted by timestamp desc:
```json
{
  "events": [
    {
      "id": "uuid",
      "ts": "2026-03-16T09:50:00Z",
      "source": "DIARY|AO|MEMORY|CRON|SYSTEM",
      "level": "info|warn|alert",
      "text": "Human readable description",
      "detail": "optional extra detail"
    }
  ],
  "total": 50
}
```

Event generation:
- DIARY: scan last diary history entries, emit one event per message
- AO: scan ao_sessions list, emit events for sessions created in last 2h
- MEMORY: scan memory_files list, emit events for files modified in last 24h
- CRON: emit events from openclaw_crons (last_run timestamps)
- SYSTEM: emit system health (memory %, docker status, uptime)

All data comes from existing sources already polled by /api/status — no new data
sources needed.

### New endpoint: GET /api/wired/stream (SSE)
Server-Sent Events stream that pushes new events as they arrive.
Poll cycle: every 5 seconds, compare with last state, emit diffs.

### wired.js
- WiredScreen class
- Connects to /api/wired/stream via EventSource (SSE)
- Falls back to polling /api/wired every 5s if SSE fails
- Renders events as terminal rows with source badge, timestamp, text
- Color coding: DIARY=orange, AO=cyan, MEMORY=purple, CRON=green, SYSTEM=yellow/red
- New events slide in from top with subtle fade animation
- Max 100 events in DOM (trim oldest)
- Auto-scroll to top on new event (unless user has scrolled down)
- File code: "Wrd001"

## Visual Style
- Event rows: dark background, left border color = source color
- Source badge: 6-char monospace label (DIARY, AO-SES, MEMORY, CRON, SYSTAT)
- Timestamp: right-aligned, dim gray, relative ("2m ago")
- Level WARN: yellow text, ALERT: red flash
- Live indicator: blinking green dot when stream active
- CRT scanline effect from existing style.css
